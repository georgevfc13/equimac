================================================================================
FLUJO MVC EN ACCIÓN
================================================================================

CREAR NUEVO PRODUCTO:
━━━━━━━━━━━━━━━━━━━━━━

1. Usuario hace clic en "+ Nuevo Producto"
   └─ Navega a: index.php?accion=formulario

2. index.php (ENRUTADOR)
   └─ Identifica acción "formulario"
   └─ Instancia InventarioController

3. InventarioController::formulario()
   ├─ Obtiene categorías del Modelo
   ├─ Renderiza views/formulario.php
   └─ Usuario completa formulario

4. Usuario envía formulario
   └─ POST a: index.php?accion=guardar

5. index.php (ENRUTADOR)
   └─ Identifica acción "guardar"

6. InventarioController::guardar()
   ├─ Obtiene datos de $_POST
   ├─ Valida entrada
   ├─ Verifica código único
   └─ Llama Inventario::crear($datos)

7. Inventario::crear() (MODELO)
   ├─ Prepara INSERT query
   ├─ Ejecuta con MySQLi
   └─ Retorna resultado

8. InventarioController (continúa)
   ├─ Renderiza views/lista.php
   ├─ Pasa productos actualizados
   └─ Muestra mensaje "Éxito"

9. Navegador
   └─ Muestra tabla con nuevo producto

EDITAR PRODUCTO:
━━━━━━━━━━━━━━━

Similar a crear, pero:
├─ InventarioController::formulario() también trae producto existente
├─ InventarioController::guardar() llama actualizar() en lugar de crear()
└─ Código no se puede editar

ELIMINAR PRODUCTO:
━━━━━━━━━━━━━━━━

1. Usuario hace clic en "🗑 Eliminar"
   └─ Confirma acción en prompt()

2. Navega a: index.php?accion=eliminar&id=1

3. InventarioController::eliminar()
   ├─ Obtiene id de $_GET
   ├─ Llama Inventario::eliminar($id)
   └─ Renderiza lista actualizada

4. Producto desaparece de tabla
